1.  - You will likely create first your local branch, then push it into the remote and create the pull request. I do always delete both, leaving everything clean and neat for other developers.
2.  With most of the Git UI this is done after the code review automatically (so whoever finally approves the pull request also merges the branch)
3.  Tags is an excellent way to keep the product modular between merges. However, when you are merging something into develop might be a hotfix — you also want to tag this node of the git tree.
