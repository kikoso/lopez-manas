Last March, I enrolled in the Windows Phone Developer Program. At the past <a href="http://www.mobileworldcongress.com/index.html" target="_blank">MWC</a> in Barcelona, I met a representative of <a href="http://www.nokia.com/" target="_blank">Nokia</a> who informed me about the program. In exchange for 2 applications for WPhone developed within 3 months, developers will get a Nokia Lumia 800. Not bad :-).

&nbsp;

After successfully applying for the program, I finally published some applications for Windows Phone. These are my series of Intense Languages: <a href="http://www.windowsphone.com/es-ES/apps/415ea147-24e3-4632-aa6c-dbfbf2dab391" target="_blank">Intense Esperanto</a>, <a href="http://www.windowsphone.com/es-ES/apps/c5e81b9e-4688-4fb4-8e74-ec9fc41ed4d2" target="_blank">Intense German</a>, <a href="http://www.windowsphone.com/es-ES/apps/0e67ab01-c9e6-4358-9d1a-e1bade44ea05" target="_blank">Intense Dutch</a>, <a href="http://www.windowsphone.com/es-ES/apps/67148ec4-59ba-43a1-9136-f2486bafc9f0" target="_blank">Intense Spanish</a>, Intense Chinese and <a href="http://www.windowsphone.com/es-ES/apps/ff67a79f-ae36-4655-ae4d-a17cd5317040" target="_blank">Intense Latin</a>.

&nbsp;

My experience developing for WPhone can be qualified as positive. Despite the obligation to develop under Windows (which in my case means that I need to install a virtual machine), the set of tools provided by Microsoft is smooth, easy to use and very intuitive. Having a little bit of experience with previous versions of Visual Studio, the adaptation to the WPhone SDK couldn't have been easier. And besides the well-known first stages curve to publish in the Market (when independent developers can really make money), it seems that the ecosystem of applications is still very little in comparison with other well-established markets (<a href="https://play.google.com/store" target="_blank">Android</a> Market and <a href="http://itunes.apple.com/es/genre/ios/id36?mt=8" target="_blank">AppleStore</a> mainly).
